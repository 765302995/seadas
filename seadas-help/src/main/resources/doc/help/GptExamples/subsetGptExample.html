These are some examples of the SeaDAS Subset tool run from the command line using GPT (graph processing tool).

Note: when processing large files with the Subset tool you may likely need to increase your memory allocation. To do this edit the file gpt.sh.


Example 1: Geographic Region Crop


This uses the SeaDAS Subset tool in GPT (command line) mode to geographically crop a single input level-2 file into a single mapped output file. The level-2 file was imaged January 23 2017 over the Gulf of Mexico by the MODIS Aqua instrument. A parameter properties "par" file is used to store the variables needed for the xml graph file.

Note: you can readily modify these parameters in this "par" file to adapt to your own specific criteria.

Command Line Entry 

gpt geoCrop.xml -p geoCropGulfMexico.par -Ssource=A2017023193500.L2_LAC_OC.nc -t A2017023193500_geoCropGulfMexico.dim

File = geoCrop.xml 

<graph id="GeoCropGraph">
  <version>1.0</version>

  <node id="GeoCropNode">
    <operator>Subset</operator>
    <sources>
        <source>${source}</source>
    </sources>

    <parameters>
        <geoRegion>POLYGON((${west} ${north}, ${east} ${north}, ${east} ${south}, ${west} ${south}, ${west} ${north}))</geoRegion>
    </parameters>

  </node>
</graph>

File = geoCropGulfMexico.par 

north=31.0
south=17.5
west=-98.5
east=-80.0

Example 2: Pixel Area Crop


This uses the SeaDAS Subset tool in GPT (command line) mode to subset based not on geographic coordinates, but directly on the native pixels of the input file. Here we will crop off 240 pixels at both the beginning and end of the scan. In other words the easternmost and westernmost pixels. One reason for doing this might be to eliminate high sensor zenith angle (low resolution) pixels from the scene. For a standard MODIS level-2 1km LAC file, this approximates the removal of pixels above a 40 degree sensor zenith angle. The level-2 file was imaged January 23 2017 over the Gulf of Mexico by the MODIS Aqua instrument. A parameter properties "par" file is used to store the variables needed for the xml graph file.

Note: you can readily modify these parameters in this "par" file to adapt to your own specific criteria.

Command Line Entry 

gpt pixelCrop.xml -p pixelCrop.par -Ssource=A2017023193500.L2_LAC_OC.nc -t A2017023193500_pixelCrop.dim

File = pixelCrop.xml 

<graph id="PixelCropGraph">
  <version>1.0</version>

  <node id="PixelCropNode">
    <operator>Subset</operator>
    <sources>
        <source>${source}</source>
    </sources>

    <parameters>
        <region>
          <x>${x}</x>
          <y>${y}</y>
          <width>${width}</width>
          <height>${height}</height>
        </region>
    </parameters>

  </node>
</graph>

File = pixelCrop.par 

x=240
y=0
width=873
height=2030

Example 3: Full Scene Subsample


This uses the SeaDAS Subset tool in GPT (command line) mode to generate a subsampled copy of an input file. In this case, we will subsample the input file by a factor of two evenly in both the x and y direction.

Note: this is not a statistical mean based downsampling, but an incremental walk through subsampling of the pixels, throwing away the in-between skipped over pixels.

Caution: there is currently a bug which prevents usage of the boolean parameter "fullSwath" which would ideally be used when retaining the full scene dimensions of the original file. The workaround here is to specifically define the pixel crop dimensions for the full input file as shown in this example.

Command Line Entry 

gpt subsample.xml -p subsample.par -Ssource=A2017023193500.L2_LAC_OC.nc -t A2017023193500_subsample.dim

File = subsample.xml 

<graph id="SubsampleGraph">
  <version>1.0</version>

  <node id="SubsampleNode">
    <operator>Subset</operator>
    <sources>
        <source>${source}</source>
    </sources>

    <parameters>
        <region>
            <x>${x}</x>
            <y>${y}</y>
            <width>${width}</width>
            <height>${height}</height>
        </region>
        <subSamplingX>${subsampling}</subSamplingX>
        <subSamplingY>${subsampling}</subSamplingY>
    </parameters>

  </node>
</graph>

File = subsample.par 

subsampling=2
x=0
y=0
width=1354
height=2030

Example 4: Full Scene Band Subset


This example generates an output file which is essentially a copy of the original file, but retaining only a few select bands.

Caution: there is currently a bug which prevents usage of the boolean parameter "fullSwath" which would ideally be used when retaining the full scene dimensions of the original file. The workaround here is to specifically define the pixel crop dimensions for the full input file as shown in this example.

Command Line Entry 

gpt bandSubset.xml -p bandSubset.par -Ssource=A2017023193500.L2_LAC_OC.nc -t A2017023193500_bandSubset.dim

File = bandSubset.xml 

<graph id="BandSubsetGraph">
  <version>1.0</version>

  <node id="BandSubsetNode">
    <operator>Subset</operator>
    <sources>
        <source>${source}</source>
    </sources>

    <parameters>
        <bandNames>${bands}</bandNames>
        <region>
            <x>${x}</x>
            <y>${y}</y>
            <width>${width}</width>
            <height>${height}</height>
        </region>
    </parameters>

  </node>
</graph>

File = bandSubset.par 

bands=chlor_a,Kd_490,Rrs_443,Rrs_547
x=0
y=0
width=1354
height=2030
