Map Gridlines

Map Gridlines have been significantly revised for SeaDAS 7.1 with a lot of new parameter option choices for the user. Below are some examples.

Editing parameters of the map gridlines - To edit any of the parameters click on the Layer Editors icon . This will enable you to edit whichever layer is currently selected. If the Map Gridline layer does not show up in the Layer Editors window, then toggle the Map Gridlines button off then on, this will make the Map Gridline layer the selected layer. The Layer Manager window is another place where you may select a layer to be the selected layer.

The following image shows the user editable components of the map gridlines. Most fields are pretty self-explanatory.



Gridline Spacing - The latitude and longitude step sizes may be set directly where the value represent degrees between the gridlines. Setting this field to zero results in Auto-Sizing. The Auto-Sizing algorithm will set the gridline spacing to a reasonable value such that there are about 3-5 lines. Also note that lines are placed at the nearest most sensible degree value.

Font Sizing - The user font settings are in term of points rather than pixels. This enables a single user font setting and default to work well with files of any resolution. Internally the font sizes are actually tied to the pixel sizes and there is a conversion of points to pixels being made.

Low Resolution (in pixel space) Problems - For files with an extremely low resolution image, less than about (100 x 100) pixels resolution, you may expect to see problems with the map gridlines, such as missing labels.

Exporting an image with gridlines - When exporting an image as a pixel-based file (for instance a png file), the gridline labels font text will of course be bounded and affected by the resolution of the exported file. To insure that the text looks good you will want to make sure the exported file is high resolution. To do this, right click on the image in SeaDAS then select "Export Image". To the left in the window which pops up, there are the options "Visible region" and "Full scene".

"Full scene" will export the image at the exact image resolution (it will not include anything outside the image - like gridlines labels, etc.). The fields below under "Image Dimension" are now tied to the number of pixels in the band. Even if you have chosen the labels to be inside, the image resolution will most likely adversely affect the look of the font.

"Visible region" will export based on the bounds defined by the image viewer window. This will enable the user to get the gridline labels into the exported file. The fields below under "Image Dimension" are now tied to the number of pixels in the viewer window on your monitor. For a publication quality image you will want to increase this. (Note: right now we haven't added in a fixed ratio feature so you will have to independently adjust both the "Width" and the "Height" fields by the same percentage. I find it easiest to increase by a factor of ten (just tack on a "0" to both the "Width" and "Height" fields.

User Preferences For this release most of the preferences have been removed, but decent automated defaults are being used which are dynamic to the image resolution. We do plan on updating the user preferences capabilities to include all the new options in a future release.
